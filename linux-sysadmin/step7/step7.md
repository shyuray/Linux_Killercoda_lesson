# 第 7 章｜我們中招了嗎
<br>

## 為什麼要學這個
<br>

新聞說某個軟體有漏洞，主管問你：「我們的機器有沒有受影響？」

這個問題聽起來很難，實際上只要回答兩件事：**這台機器有沒有裝那個軟體？裝的是哪一個版本？**

漏洞公告都會寫「影響 X 版以下」。你手上有版本號，就能直接比對。

---
<br>

## 核心觀念
<br>

Linux 上的軟體不是自己去網站下載安裝，而是交給**套件管理系統**統一處理。Ubuntu 上的工具叫 `apt`。

```text
【圖：套件管理系統在管什麼】

   官方倉庫（網路上）              你的機器
  ┌──────────────────┐         ┌──────────────────┐
  │  nginx  1.24.0-2 │         │  nginx  1.24.0-1 │
  │  python3 3.12.3  │  ◀────▶ │  python3 3.12.3  │
  │  curl   8.5.0-2  │         │  curl   8.5.0-2  │
  └──────────────────┘         └──────────────────┘
       Candidate                    Installed
     倉庫裡最新的版本              目前實際裝的版本

     兩邊一樣  ──▶  已經是最新
     裝的比較舊 ──▶  有更新沒裝，可能存在已知漏洞
```

有兩組指令很容易混淆，先分清楚：

| 指令 | 它實際做什麼 | 結果 |
| :--- | :--- | :--- |
| `apt update` | 去倉庫拿一份最新的清單 | **軟體一個都沒變** |
| `apt upgrade` | 照著清單，下載並安裝新版本 | 軟體真的升級了 |

順序永遠是先 `update` 再 `upgrade`。反過來的話，你會照著一份過期的清單去升級。

---
<br>

## 這一章的指令
<br>

| 指令 | 它回答什麼 |
| :--- | :--- |
| `apt policy 套件名` | 目前裝的是哪一版，倉庫裡最新的是哪一版 |
| `dpkg -l \| grep 套件名` | 這個套件裝了沒，版本多少 |
| `apt list --upgradable` | 有哪些套件可以升級 |
| `which 指令名` | 這個指令的執行檔放在哪裡 |

---
<br>

## 動手做
<br>

**第一步：查一個一定裝了的套件。**

`apt policy python3`{{execute}}

**第二步：查一個可能沒裝的套件，對照差別。**

`apt policy nginx`{{execute}}

*→ 如果沒裝，`Installed` 那一行會顯示 `(none)`。*

**第三步：看看有哪些套件可以升級。**

`apt list --upgradable`{{execute}}

*→ 這張清單就是待辦事項。清單是空的也是一個有效的結論。*

## 判讀輸出
<br>

### 1. `apt policy` 你只要看兩行

```text
python3:
  Installed: 3.12.3-0ubuntu2        ← 你現在裝的版本
  Candidate: 3.12.3-0ubuntu2.1      ← 倉庫裡最新的版本
  Version table:
     ...
```

| 情況 | 結論 |
| :--- | :--- |
| `Installed` 等於 `Candidate` | 已經是倉庫裡最新的版本 |
| `Installed` 舊於 `Candidate` | **有更新沒裝，可能存在已知漏洞** |
| `Installed: (none)` | 這台機器沒有裝這個套件，漏洞公告跟你無關 |

### 2. `dpkg -l` 開頭那兩個字母

```text
ii  python3   3.12.3-0ubuntu2   amd64   interactive high-level language
▲▲
```

| 你看到 | 代表 | 排查時的意義 |
| :--- | :--- | :--- |
| `ii` | 已安裝、設定完成 | 正常運作中 |
| `rc` | 程式已移除，但設定檔還留著 | 有人只做了移除沒有清乾淨。**那個殘留的設定檔值得看一眼** |

### 3. 這些輸出代表什麼結論

| 情況 | 結論 |
| :--- | :--- |
| 漏洞公告說「1.24 以下受影響」，你查到裝的是 1.20 | 受影響，要排時間升級 |
| 查到 `Installed: (none)` | 沒有裝，不受影響。**這個答案跟「已經是最新版」一樣有價值** |
| `apt list --upgradable` 是空的 | 系統已經是最新狀態 |
| `apt list --upgradable` 有一長串 | 這台機器很久沒有更新過 |

> **升級不能隨手做。**
>
> 升級可能會重新啟動服務、改變設定檔、甚至讓相依的程式無法運作。正式環境要先確認影響範圍、排定維護時間。
>
> **升級完成之後，一定要回頭用第 5 章的方法確認服務還在執行。**
