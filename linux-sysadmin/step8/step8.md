# 第 8 章｜兩個程式要用同一個連接埠
<br>

## 為什麼要學這個
<br>

報表系統啟動不了。你用第 5 章的方法查了狀態，是 `failed`；用第 6 章的方法讀了紀錄，寫著 `Address already in use`。

設定沒有錯，權限也沒有問題。**問題在於它要用的那個位置，已經被另一個程式佔住了。**

這一章要學會的是：看出這台機器對外開了哪些位置、每一個位置是誰在用。

---
<br>

## 核心觀念
<br>

一台伺服器只有一個網路位址，但上面同時跑著好幾個服務。外面來的請求要怎麼知道該交給誰？靠的是**連接埠**。

```text
【圖：連接埠是同一個位址上的不同門牌】

              srv-app01
       ┌───────────────────────┐
       │                       │
  ───▶ │  :22    ──▶  SSH 遠端連線
  外面 │                       │
  的請 │  :8080  ──▶  訂單查詢系統
  求   │                       │
       │  :9000  ──▶  報表系統   ◀── 這個位置已經有別人了
       │                       │
       └───────────────────────┘

    一個連接埠同一時間只能被一個程式佔用。
    第二個程式想用同一個號碼，就會啟動失敗。
```

| 常見連接埠 | 通常是誰在用 |
| :--- | :--- |
| 22 | SSH 遠端連線 |
| 80 | 網頁（未加密） |
| 443 | 網頁（加密） |
| 3306 | MySQL 資料庫 |

---
<br>

## 這一章的指令
<br>

| 指令 | 它回答什麼 |
| :--- | :--- |
| `ss -tulpn` | 這台機器開了哪些連接埠，各是哪個程式在用 |
| `ss -tulpn \| grep 埠號` | 只看某一個連接埠 |
| `ufw status` | 防火牆現在是啟用還是停用，規則有哪些 |

> **`ss -tulpn` 的四個參數：** `-t` 看 TCP，`-u` 看 UDP，`-l` 只看正在等待連線的，`-n` 直接顯示號碼不要轉成名稱。
>
> 另外還有一個 `-p`，用來顯示是哪一個程式開的。**這一項需要管理員權限才看得到。**

---
<br>

## 動手做
<br>

**第一步：先確認服務是什麼狀態，以及紀錄怎麼說。**

`systemctl status report-api --no-pager`{{execute}}

`journalctl -u report-api -n 10 --no-pager`{{execute}}

**第二步：查那個連接埠現在是誰在用。**

`ss -tulpn | grep 9000`{{execute}}

**第三步：看整台機器開了哪些連接埠。**

`ss -tulpn`{{execute}}

**第四步：查防火牆狀態。**

`ufw status`{{execute}}

---
<br>

## 判讀輸出
<br>

### 1. `ss -tulpn` 你只要看兩欄

```text
Netid  State   Recv-Q  Send-Q   Local Address:Port    Peer Address:Port   Process
tcp    LISTEN  0       5        0.0.0.0:9000          0.0.0.0:*           users:(("python3",pid=812,...))
                                    ▲    ▲                                        ▲
                              監聽範圍  連接埠                              是哪個程式
```

**`Local Address` 那一欄決定外面連不連得到：**

| 你看到 | 意思 | 外面連得到嗎 |
| :--- | :--- | :--- |
| `0.0.0.0:8080` | 監聽所有網路介面 | **連得到** |
| `[::]:8080` | 同一個連接埠的 IPv6 版本 | **連得到** |
| `127.0.0.1:3306` | 只監聽本機回送位址 | 連不到，只有這台機器自己能用 |

**這個差別在資安上很重要。** 一個只讓本機使用的資料庫，跟一個對全世界開放的資料庫，風險完全不同，但它們在 `ss` 的輸出裡只差幾個字。

### 2. `ufw status` 的兩種結果

| 你看到 | 結論 |
| :--- | :--- |
| `Status: active` 加上一串規則 | 防火牆正在運作，只有列出來的連接埠通得過 |
| `Status: inactive` | **防火牆沒有在運作。規則就算寫好了也不會生效** |

規則存在跟規則生效是兩件事。**只看到規則清單就以為有防護，是很常見的誤判。**

### 3. 這些輸出代表什麼結論

| 情況 | 結論 |
| :--- | :--- |
| 紀錄寫 `Address already in use` | 有別的程式先佔住了那個連接埠 |
| `ss` 查到該埠的 Process 不是你要的服務 | 找到佔用者了。**先確認它是什麼、能不能停，不要直接砍** |
| `ss` 查不到該埠 | 沒有人在用，問題不在連接埠，回頭查設定 |
| 有連接埠開在 `0.0.0.0` 但你不知道那是什麼服務 | 這是資安風險，要查清楚 |

## 任務
<br>

| # | 找出什麼 |
| :--- | :--- |
| 1 | `report-api` 啟動失敗的原因，紀錄裡是哪一句話說的 |
| 2 | 9000 這個連接埠現在被哪一個程式佔住，它的程序編號是多少 |
| 3 | 這台機器上有哪些連接埠是對外開放的，哪些只限本機使用 |

第 3 題要從 `Local Address` 那一欄判斷，不是從連接埠號碼判斷。
