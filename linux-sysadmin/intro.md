# Linux 系統管理：從接手一台伺服器開始
<br>

## 什麼時候會用到 Linux
<br>

Linux 伺服器沒有圖形介面。你不能像在自己的電腦上那樣，打開檔案總管看看裡面有什麼、對著檔案按右鍵看它的內容。伺服器上發生的每一件事，都不會主動顯示在任何畫面上。

**你只能用指令去問系統，而系統回給你的是一大片文字，答案就藏在其中幾個字裡面。**

所以，當下面這些狀況發生的時候，你需要的是知道該問哪一個問題、以及怎麼從那片文字裡把答案挑出來。

| 狀況 | 你會被問的問題 |
| :--- | :--- |
| 內部系統打不開，全公司都在等 | 是系統本身出了狀況，還是有人把它關掉了？從什麼時候開始的？ |
| 同事打不開某個檔案，你自己卻開得起來 | 同樣一個檔案，為什麼他沒有權限？ |
| 有同事離職 | 他的帳號還能不能登入這台機器？ |
| 硬碟空間用完，系統開始出錯 | 空間是被什麼佔掉的？哪些可以刪？ |
| 新聞說某個軟體有漏洞 | 我們裝的是哪一個版本？有沒有受影響？ |

## 這台機器
<br>

你接手了公司的一台伺服器 `srv-app01`。這台機器上有三套公司內部每天在使用的系統：

- 訂單查詢系統，業務部每天都會用到
- 部門共用的檔案目錄
- 每月自動產出的報表

前一位負責這台機器的同事已經離職，沒有留下任何交接文件。

## 十個章節
<br>

| # | 章節 | 你會學會 |
| :--- | :--- | :--- |
| 1 | 我在哪裡，這裡有什麼 | 在系統中定位，讀懂檔案清單 |
| 2 | 從一堆檔案裡找到目標 | 依檔名搜尋、依內容搜尋 |
| 3 | 檔案在，但打不開 | 讀懂權限，判斷誰可以讀寫 |
| 4 | 誰能登入這台機器 | 清查帳號，處理離職同事留下的帳號 |
| 5 | 系統打不開了 | 判斷服務狀態，讓它恢復執行 |
| 6 | 昨晚發生了什麼 | 從紀錄裡找出原因 |
| 7 | 我們中招了嗎 | 查版本，判斷有沒有已知漏洞 |
| 8 | 兩個程式要用同一個連接埠 | 查看連接埠與防火牆 |
| 9 | 檔案寫不進去了 | 查磁碟空間與掛載狀況 |
| 10 | 只有症狀，沒有線索 | 綜合前九章，自己找出原因 |
