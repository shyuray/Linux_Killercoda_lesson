# 第 3 章｜檔案在，但打不開
<br>

## 為什麼要學這個
<br>

同事說他打不開部門共用區裡的報表，你自己去看，卻讀得出來。

檔案就在那裡，路徑也沒錯，兩個人拿到的結果卻不一樣。**差別在於系統是照「誰要來讀」決定要不要放行的。**

這一章要學會的判斷是：拿到一個檔案，看出誰可以讀、誰可以寫、誰連進去都不行。

---
<br>

## 核心觀念
<br>

`ls -l` 最前面那十個字元，就是這個檔案的通行規則。

```text
【圖：權限的十個字元怎麼拆】

     -   rw-   r--   r--
     │    │     │     │
     │    │     │     └── others：其他所有人
     │    │     └──────── group：所屬群組的成員
     │    └────────────── owner：檔案的擁有者
     └─────────────────── 類型：- 是檔案，d 是目錄

     每一組都是三格，順序固定是 r w x
     ┌─────┬──────────────┬──────────────────┐
     │ 字母 │  對檔案而言   │   對目錄而言      │
     ├─────┼──────────────┼──────────────────┤
     │  r  │  可以讀內容   │  可以列出裡面有什麼 │
     │  w  │  可以修改     │  可以新增或刪除檔案 │
     │  x  │  可以執行     │  可以進入這個目錄   │
     │  -  │  沒有這個權限 │  沒有這個權限      │
     └─────┴──────────────┴──────────────────┘
```

**系統判斷的順序是：先看你是不是擁有者，是的話就用擁有者那組，不再往下看；不是的話看你在不在群組裡；都不是，才用其他人那組。**

---
<br>

## 這一章的指令
<br>

| 指令 | 它回答什麼 |
| :--- | :--- |
| `ls -l 檔案` | 這個檔案的權限、擁有者、群組 |
| `ls -ld 目錄` | **這個目錄本身**的權限（不加 `-d` 會變成列出目錄裡面的東西） |
| `chmod 權限 對象` | 修改權限 |
| `chown 帳號:群組 對象` | 修改擁有者與所屬群組 |

> **`ls -l` 和 `ls -ld` 差一個字母，意思完全不同。**
>
> `ls -l 目錄` 是列出這個目錄**裡面**有什麼。
>
> `ls -ld 目錄` 是看這個目錄**自己**的權限。排查權限問題要看的是後者。

## 動手做
<br>

**第一步：先看那個檔案本身的權限。**

`ls -l /srv/share/reports/q2_summary.csv`{{execute}}

*→ 看 others 那三格。這個檔案對所有人都開放讀取。*

**第二步：再看它所在的目錄。**

`ls -ld /srv/share/reports`{{execute}}

*→ 這一步最容易被跳過，也是問題真正的所在。*

**第三步：對照另一個目錄。**

`ls -ld /srv/share/sales`{{execute}}

*→ 這個目錄的權限跟上一個完全不同。哪一種比較危險？*

## 判讀輸出
<br>

### 1. 兩個輸出擺在一起看

```text
-rw-r--r--   root  appteam   q2_summary.csv     ← 檔案：其他人可以讀
drwxr-x---   root  appteam   reports            ← 目錄：其他人連進去都不行
    ▲▲▲
    others 這三格是 --- ，沒有 x
```

**要讀到一個檔案，你必須先能進入它所在的目錄。** 進入目錄需要的是目錄的 `x` 權限。

目錄沒有 `x`，檔案設得再開放都沒有用，因為外面那道門是關著的。**這是權限問題最常見、也最容易被忽略的一層。**

### 2. 常見的權限組合代表什麼

| 你看到 | 拆開來是 | 代表 |
| :--- | :--- | :--- |
| `-rw-r--r--` | 擁有者讀寫，其他人唯讀 | 一般檔案最常見的設定 |
| `drwxr-xr-x` | 所有人都可以進入與列出 | 一般目錄最常見的設定 |
| `drwxr-x---` | 只有擁有者與群組可以進入 | **其他人被擋在門外** |
| `drwxrwxrwx` | 所有人都可以進入、新增、刪除 | **任何人都能改裡面的檔案，這是資安漏洞** |
| `----------` | 沒有任何人有任何權限 | 連擁有者自己都讀不到 |

### 3. 這些輸出代表什麼結論

| 情況 | 結論 |
| :--- | :--- |
| 檔案 others 有 `r`，但目錄 others 沒有 `x` | 問題出在目錄，不在檔案 |
| 目錄權限是 `777` | 權限開得過大，任何帳號都能修改裡面的內容 |
| 你自己讀得到，別人讀不到 | 你的身分跟他不同。**判斷權限之前，先確認是誰要去讀** |
