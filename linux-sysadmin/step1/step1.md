# 第 1 章｜我在哪裡，這裡有什麼
<br>

## 為什麼要學這個
<br>

Linux 沒有可以用滑鼠點開的資料夾。你要知道自己站在哪個位置、這個位置裡有哪些檔案，全部都得用指令去問。

最常見的事故是這樣發生的：有人以為自己在測試目錄底下，執行了刪除，結果刪掉的是正式資料。**刪除沒有復原鍵。**

動手之前先確認位置，這個習慣能擋掉大部分無法挽回的錯誤。

---
<br>

## 核心觀念
<br>

Linux 上所有的檔案都掛在同一棵樹上，最上層是 `/`。你隨時都站在這棵樹的某一個位置。

```text
【圖：檔案樹與目前位置】

                    /
                    │
     ┌──────────┬───┴────┬──────────┐
     │          │        │          │
   /etc       /home    /var       /srv
                │        │          │
                │      /var/log   /srv/app
                │                   │
           /home/user1        ┌─────┴─────┐
                              │           │
                          /srv/app/    /srv/app/
                            config       data
                                           ▲
                                     你現在在這裡
```

所有操作都圍繞著三個問題：

| 問題 | 用什麼問 |
| :--- | :--- |
| 我現在站在哪裡 | `pwd` |
| 這個位置裡有什麼 | `ls` |
| 我要移動到別的位置 | `cd` |

---
<br>

## 這一章的指令
<br>

| 指令 | 它回答什麼 |
| :--- | :--- |
| `pwd` | 我現在在哪一個目錄 |
| `ls -l` | 這個目錄裡有什麼，以及每個項目的詳細資訊 |
| `ls -la` | 同上，但連隱藏檔一起顯示 |
| `cd 路徑` | 移動到指定的目錄 |

> 路徑有兩種寫法。以 `/` 開頭的是**絕對路徑**，不管你目前站在哪裡，它都指向同一個位置。不以 `/` 開頭的是**相對路徑**，從你目前的位置往下算。
>
> **不確定的時候用絕對路徑。**

## 動手做
<br>

**第一步：確認目前位置。**

`pwd`{{execute}}

**第二步：移動到訂單系統的資料目錄，看看裡面有什麼。**

`cd /srv/app/data`{{execute}}

`ls -l`{{execute}}

**第三步：加上 `-a`，看看有沒有原本被藏起來的檔案。**

`cd /srv/app`{{execute}}

`ls -la`{{execute}}

*→ 比較兩次輸出的差別。以 `.` 開頭的檔案，預設不會顯示。*

## 判讀輸出
<br>

`ls -l` 的每一行都是同樣的七段，位置固定不變：

```text
-rw-r--r--  1  root  root  1842  Jul 23 09:14  app.conf
    ①      ②   ③     ④     ⑤        ⑥          ⑦
```

| 位置 | 內容 | 你要從這裡看出什麼 |
| :--- | :--- | :--- |
| ① | 類型與權限 | **第一個字元是類型**：`-` 表示一般檔案，`d` 表示目錄。後面九個字元是權限，第 3 章會拆解 |
| ② | 連結數 | 這一章用不到，跳過 |
| ③ | 擁有者 | 這個檔案屬於哪一個帳號 |
| ④ | 所屬群組 | 這個檔案屬於哪一個群組 |
| ⑤ | 大小 | 單位是位元組。加上 `-h` 會換成看得懂的單位 |
| ⑥ | 最後修改時間 | **排查問題時最有用的一欄** |
| ⑦ | 名稱 | 檔案或目錄的名字 |

### 這些輸出代表什麼結論

| 你看到 | 結論 |
| :--- | :--- |
| 第一個字元是 `d` | 這是目錄，可以用 `cd` 進去 |
| 第一個字元是 `-` | 這是檔案，可以用 `cat` 讀 |
| 大小是 `0` | 檔案存在，但裡面沒有內容。這通常本身就是一個問題 |
| 修改時間落在故障發生的前後 | 這個檔案值得優先檢查，因為它在事發前後被動過 |

> **一個常見的誤判：** 檔名沒有出現在清單裡，不代表它不存在。以 `.` 開頭的檔案要加上 `-a` 才看得到。

## 任務
<br>

用這一章的指令，在環境裡找出下面三件事的答案。

| # | 找出什麼 |
| :--- | :--- |
| 1 | `/srv/app` 底下有幾個目錄、幾個檔案 |
| 2 | `/srv/app/data` 裡面哪一個檔案是空的 |
| 3 | `/srv/app` 整棵目錄樹裡，最久沒有被修改的是哪一個檔案 |

第三題需要你進到不同的目錄，比較各個檔案的修改時間。
