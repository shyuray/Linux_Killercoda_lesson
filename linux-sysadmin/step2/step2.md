# 第 2 章｜從一堆檔案裡找到目標
<br>

## 為什麼要學這個
<br>

同事說：「公司換了資料庫主機，舊的位址要全部改掉。」

一台伺服器上有幾萬個檔案，寫著舊位址的可能只有一個，也可能有五個。你不可能一個目錄一個目錄點進去看。

**Linux 提供兩種搜尋方式，差別在於你手上有什麼線索。**

---
<br>

## 核心觀念
<br>

```text
【圖：兩種搜尋方式的分工】

你手上有的線索                用哪個工具              它回你什麼
──────────────────────────────────────────────────────────────
知道檔案叫什麼名字     ──▶     find      ──▶     檔案的完整路徑
   例：db.conf

知道檔案裡面           ──▶     grep      ──▶     哪一個檔案的
寫著什麼內容                                       哪一行命中
   例：192.168.10.50
```

| 你的處境 | 用哪一個 |
| :--- | :--- |
| 我知道檔名，但不知道它在哪個目錄 | `find` |
| 我不知道檔名，只知道裡面寫了什麼 | `grep` |

---
<br>

## 這一章的指令
<br>

| 指令 | 它回答什麼 |
| :--- | :--- |
| `find 起點 -name "檔名"` | 從起點往下找，哪些檔案叫這個名字 |
| `find 起點 -type f` | 從起點往下列出所有檔案（`-type d` 則是目錄） |
| `grep "字串" 檔案` | 這個檔案裡，哪幾行含有這個字串 |
| `grep -r "字串" 目錄` | 這個目錄底下的所有檔案，哪一個含有這個字串 |

> **`grep` 的參數順序很容易寫反：要找的字串在前，檔案或目錄在後。**
>
> 寫反的話，系統會把你要找的字串當成檔名去開，然後告訴你檔案不存在。

---
<br>

## 動手做
<br>

**第一步：先看看設定檔目錄裡有多少東西。**

`ls -R /srv/app/config`{{execute}}

*→ `-R` 會連子目錄一起列出來。你會看到檔案分散在好幾層。*

**第二步：依檔名搜尋。**

`find /srv/app -name "db.conf"`{{execute}}

**第三步：依內容搜尋。**

`grep -r "192.168.10" /srv/app/config`{{execute}}

*→ 比較第二步與第三步的結果。兩種搜尋方式找到的檔案，數量不一樣。*

---
<br>

## 判讀輸出
<br>

### 1. `find` 回給你的是完整路徑

```text
/srv/app/config/legacy/db.conf
```

`find` 只回答一件事：符合條件的檔案在哪裡。它不會告訴你裡面寫了什麼。

| 你看到 | 結論 |
| :--- | :--- |
| 一行路徑 | 找到唯一一個符合的檔案 |
| 好幾行路徑 | 有多個同名檔案分散在不同目錄，**每一個都要確認** |
| 沒有任何輸出 | 這個起點底下沒有符合的檔案。**沒有輸出就是結果，不是指令失敗** |

### 2. `grep -r` 回給你的是「檔案路徑加冒號加命中的那一行」

```text
/srv/app/config/legacy/db.conf:db_host = 192.168.10.50
└──────────── 檔案在哪 ────────┘└──── 這一行寫了什麼 ────┘
                              ▲
                            冒號是分隔點
```

**冒號左邊是檔案路徑，右邊是命中的那一整行原文。**

| 你看到 | 結論 |
| :--- | :--- |
| 命中的行數比 `find` 找到的檔案多 | 有些檔案的名字不叫 `db.conf`，但裡面同樣寫著舊位址 |
| 同一個檔案出現好幾行 | 這個檔案裡有多處都要改 |
| 出現 `Permission denied` | 這個目錄你沒有權限進去讀，第 3 章會處理 |

> **這一章要建立的判斷：** 依檔名找到的結果，跟依內容找到的結果，經常不一樣。
>
> 只用其中一種搜尋就下結論，會漏掉東西。**要改設定之前，兩種都要跑過。**

## 任務
<br>

| # | 找出什麼 |
| :--- | :--- |
| 1 | `/srv/app/config` 底下總共有幾個副檔名是 `.conf` 的檔案 |
| 2 | 哪些檔案裡面寫著 `192.168.10` 開頭的位址，各自寫的是哪一個位址 |
| 3 | 第 2 題找到的檔案裡，有沒有哪一個是第 1 題的檔名搜尋抓不到的 |

第 3 題是這一章的重點。
