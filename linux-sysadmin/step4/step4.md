# 第 4 章｜誰能登入這台機器
<br>

## 為什麼要學這個
<br>

前一位負責這台機器的同事已經離職半年，沒有人記得他的帳號有沒有停用。

**離職者的帳號如果還能登入，公司對這台機器就失去了控制。** 你不知道誰在用它、不知道它做過什麼。

這一章要學會的是：列出這台機器上所有的帳號，判斷哪些真的可以登入、哪些擁有管理員權限。

---
<br>

## 核心觀念
<br>

所有帳號都記在 `/etc/passwd` 這個檔案裡，一行一個帳號，用冒號分成七欄。

```text
【圖：/etc/passwd 的七個欄位】

  wchen : x : 1002 : 1005 : Wu Chen : /home/wchen : /bin/bash
    │     │    │      │       │           │            │
    ①     ②    ③      ④       ⑤           ⑥            ⑦

  ① 帳號名稱
  ② 密碼欄（現在固定是 x，真正的密碼放在 /etc/shadow）
  ③ 使用者編號 UID  ◀── 系統判斷權限看的是這個數字，不是名字
  ④ 主要群組編號 GID
  ⑤ 說明欄（通常放全名）
  ⑥ 家目錄
  ⑦ 登入後執行的程式  ◀── 這一欄決定這個帳號能不能登入
```

**第七欄是這一章的重點：**

| 第七欄的值 | 代表 |
| :--- | :--- |
| `/bin/bash` | 登入後會得到一個可以打指令的環境，**這個帳號能登入** |
| `/usr/sbin/nologin` | 印一句「不開放登入」然後立刻結束連線 |
| `/bin/false` | 不印任何訊息，直接以失敗結束 |

服務用的帳號會刻意設成後面兩種：它需要擁有檔案、需要拿來執行程式，但不需要有人坐在前面打字。**把登入這個能力拿掉，攻擊面就少一塊。**

---
<br>

## 這一章的指令
<br>

| 指令 | 它回答什麼 |
| :--- | :--- |
| `cat /etc/passwd` | 這台機器上有哪些帳號 |
| `grep "字串" /etc/passwd` | 只列出符合條件的那幾行 |
| `id 帳號` | 這個帳號的編號與所屬群組 |
| `getent group sudo` | 誰在管理員群組裡 |
| `usermod -L 帳號` | 鎖定這個帳號，讓它無法用密碼登入 |

> **發現可疑或該停用的帳號時，先鎖定，不要刪除。**
>
> 帳號一旦刪掉，它做過什麼、擁有哪些檔案，就再也查不出來了。鎖定可以立刻阻止登入，同時保留所有紀錄。

---
<br>

## 動手做
<br>

**第一步：只看能登入的帳號。**

`grep "/bin/bash" /etc/passwd`{{execute}}

*→ 用第七欄當關鍵字，一次篩掉所有不能登入的服務帳號。*

**第二步：看誰在管理員群組裡。**

`getent group sudo`{{execute}}

*→ 最後一欄用逗號隔開的，就是這個群組的成員。*

**第三步：查一個服務帳號，對照差別。**

`grep "app-report" /etc/passwd`{{execute}}

`id app-report`{{execute}}

## 判讀輸出
<br>

### 1. `getent group sudo` 的四個欄位

```text
sudo : x : 27 : sysadmin,wchen
  │     │   │        │
  │     │   │        └── 成員清單，用逗號隔開
  │     │   └─────────── 群組編號
  │     └─────────────── 密碼欄，不使用
  └───────────────────── 群組名稱
```

**在 Ubuntu 上，屬於 `sudo` 群組等於可以取得管理員權限。** 這一行列出的每一個名字，都能對這台機器做任何事。

### 2. `id` 的輸出

```text
uid=997(app-report) gid=997(app-report) groups=997(app-report)
```

| 你看到 | 結論 |
| :--- | :--- |
| `uid` 是 1000 以上 | 這是給人使用的一般帳號 |
| `uid` 小於 1000 | 這是系統或服務用的帳號 |
| `uid=0` | **這個帳號等同於 root**。名字叫什麼不重要，系統只看數字 |
| `groups` 裡有 `sudo` | 這個帳號可以取得管理員權限 |

### 3. 這些輸出代表什麼結論

| 情況 | 結論 |
| :--- | :--- |
| 一個帳號第七欄是 `/bin/bash`，但那個人已經離職 | 這個帳號還能登入，應該鎖定 |
| 離職者的帳號同時在 `sudo` 群組裡 | 風險等級提高，因為它不只能登入，還能做任何事 |
| 服務帳號的第七欄是 `/bin/bash` | 設定有問題。服務帳號不需要登入能力 |
| 兩個不同名字的帳號，`uid` 卻相同 | 兩者的權限完全一樣。**這種設定幾乎都是刻意做出來的** |
