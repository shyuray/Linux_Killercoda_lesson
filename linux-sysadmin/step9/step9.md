# 第 9 章｜檔案寫不進去了
<br>

## 為什麼要學這個
<br>

備份沒有跑成功，紀錄裡寫著 `No space left on device`。

設定沒有動過，權限也沒有改過。**這台機器不是出了故障，是資源被用完了。**

服務需要的不只是正確的設定與權限，它還需要磁碟空間。任何一樣被用光，它都無法運作。而且把磁碟塞滿的，經常就是紀錄檔本身。

---
<br>

## 核心觀念
<br>

一台機器的磁碟不是一整塊，而是切成好幾個部分，各自掛在不同的路徑底下。

```text
【圖：一台機器有好幾個獨立的空間】

   /            19G   用了 30%   ← 系統與程式
   ├── /boot   881M   用了 15%   ← 開機檔案
   └── /srv/backup  90M  用了 92%  ← 備份區，快滿了
                          ▲
              根目錄還很空，不代表每個部分都還有空間。
              滿掉的那一個，掛在它底下的東西就寫不進去。
```

查空間有兩個工具，用途不同，順序不能顛倒：

| 工具 | 它回答什麼 | 什麼時候用 |
| :--- | :--- | :--- |
| `df` | 每一個部分**總共多大、還剩多少** | 先用它確認是哪一個滿了 |
| `du` | 某一個目錄**用掉多少** | 再用它找出是誰佔的 |

**先 `df` 定位，再 `du` 追查。** 反過來會很慢，因為你不知道該從哪個目錄開始找。

---
<br>

## 這一章的指令
<br>

| 指令 | 它回答什麼 |
| :--- | :--- |
| `df -h` | 每個部分的總量、已用、剩餘、使用率 |
| `du -sh 目錄/*` | 這個目錄底下，各個項目分別佔了多少 |
| `ls -lh 檔案` | 單一檔案的大小 |
| `truncate -s 0 檔案` | 把檔案內容清空，但保留檔案本身 |

> **`-h` 的意思是換成人看得懂的單位**，把一長串數字變成 `1.2G`、`340M`。查空間的指令幾乎都要加它。

---
<br>

## 動手做
<br>

**第一步：先確認是哪一個部分滿了。**

`df -h`{{execute}}

*→ 看 `Use%` 那一欄，同時注意 `Mounted on` 告訴你滿的是哪一個位置。*

**第二步：找出是誰佔掉的。**

`du -sh /srv/backup/*`{{execute}}

**第三步：換一個目錄，看紀錄檔佔了多少。**

`du -sh /var/log/app/*`{{execute}}

`ls -lh /var/log/app/access.log`{{execute}}

<br>

## 判讀輸出
<br>

### 1. `df -h` 的六欄

```text
Filesystem      Size  Used  Avail  Use%  Mounted on
/dev/loop0       90M   77M   6.7M   92%  /srv/backup
                                     ▲        ▲
                            你的眼睛看這欄   滿的是這個位置
```

| 欄位 | 意思 |
| :--- | :--- |
| `Size` | 這個部分總共多大 |
| `Used` / `Avail` | 用掉多少 / 還剩多少 |
| `Use%` | 使用率百分比 |
| `Mounted on` | 這個部分對應到哪一個路徑 |

**使用率到 90% 就該處理，不要等到 100%。** 到 100% 的時候，很多服務已經停了，而且你連自己想建一個暫存檔都會失敗。

### 2. `du -sh` 的輸出

```text
38M   /srv/backup/daily/db-20260720.tar.gz
38M   /srv/backup/daily/db-20260721.tar.gz
```

左邊是大小，右邊是路徑。**加上 `sort -h` 可以由小到大排，最大的會排在最後面。**

### 3. 這些輸出代表什麼結論

| 情況 | 結論 |
| :--- | :--- |
| 根目錄很空，但某個掛載點 92% | **問題出在那個掛載點**，不要看根目錄就說「還有空間」 |
| 佔空間的是備份檔 | 要判斷保留幾天，清掉超過期限的 |
| 佔空間的是紀錄檔 | 紀錄檔會無限長大，這是最常見的原因 |

> **清空一個服務正在使用的紀錄檔，要用 `truncate`，不要用 `rm`。**
>
> 服務還開著那個檔案的時候，你把它刪掉，檔名會消失，但空間不會立刻釋放，而且服務會失去它的紀錄檔。
>
> `truncate -s 0` 把內容清空、保留檔案本身，空間立刻回來，服務也能繼續往裡面寫。
