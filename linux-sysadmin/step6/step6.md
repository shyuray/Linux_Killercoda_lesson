# 第 6 章｜昨晚發生了什麼
<br>

## 為什麼要學這個
<br>

月報表沒有產出來。排程時間是凌晨，當時沒有人在。

**服務停下來的時候不會不吭聲，它一定會把「我遇到了什麼問題」寫下來。** 你的工作不是猜它為什麼停，是把它寫下來的那句話找出來讀懂。

這一章要學會的是：從幾百行紀錄裡，挑出真正說明原因的那一行。

---
<br>

## 核心觀念
<br>

紀錄分成兩個地方存放，讀它們的工具不一樣，不要拿錯。

```text
【圖：兩種紀錄，兩種讀法】

  ┌────────────────────────┐        ┌────────────────────────┐
  │  systemd 統一收集的     │        │  /var/log 底下的        │
  │  服務啟動、停止、錯誤    │        │  各程式自己寫的檔案      │
  ├────────────────────────┤        ├────────────────────────┤
  │  格式：二進位           │        │  格式：純文字           │
  │  不能用 cat 讀          │        │  可以用 cat、tail 讀    │
  ├────────────────────────┤        ├────────────────────────┤
  │  用 journalctl          │        │  用 tail、grep          │
  └────────────────────────┘        └────────────────────────┘
       服務為什麼停？                     程式自己記了什麼？
```

還有一件事能替你省下大量時間：**紀錄裡絕大部分是雜訊。** 一個服務每天寫幾千行，多數是「我正常運作中」的例行訊息。你要學的不是讀得快，是懂得過濾。

---
<br>

## 這一章的指令
<br>

| 指令 | 它回答什麼 |
| :--- | :--- |
| `journalctl -u 服務 -n 20` | 這個服務最近說了什麼（只看最後 20 行） |
| `tail -n 20 檔案` | 純文字紀錄檔的最後 20 行 |
| `grep "ERROR" 檔案` | 只撈出含有這個關鍵字的行 |
| `grep -c "ERROR" 檔案` | 只回答有幾行命中，不印內容 |

> **參數說明：** `-u` 指定服務，`-n` 限制行數，`-c` 只計數。
>
> 紀錄檔動輒上萬行，這幾個參數的用途都是縮小範圍。**沒有人從第一行開始讀。**

---
<br>

## 動手做
<br>

**第一步：問那個失敗的服務發生了什麼事。**

`journalctl -u report-job -n 20 --no-pager`{{execute}}

*→ 由下往上讀。中間會有一行直接說明原因。*

**第二步：換另一種紀錄，看訂單系統自己記了什麼。**

`tail -n 10 /var/log/app/order.log`{{execute}}

**第三步：從幾十行裡只撈出錯誤。**

`grep "ERROR" /var/log/app/order.log`{{execute}}

*→ 比較第二步與第三步。過濾之後剩下幾行？*

## 判讀輸出
<br>

### 1. 錯誤訊息之間有因果順序

```text
[原因] Failed to execute /opt/scripts/gen_report.sh: No such file or directory
         │
         ▼  找不到要執行的檔案
[結果] report-job.service: Main process exited, code=exited, status=203/EXEC
         │
         ▼  程式無法啟動，回報錯誤代碼
[結果] Failed to start Monthly Report Job.
```

**新手常常盯著最後一行的 `Failed to start` 發呆。** 那一行只告訴你「它失敗了」，不會告訴你為什麼。

**真正的原因在更上面。往上找第一個錯誤，那才是根本原因。** 你修得了第一行，修不了最後一行。

### 2. 紀錄的等級

| 等級 | 意思 | 該不該理它 |
| :--- | :--- | :--- |
| `INFO` | 例行訊息：「我開始工作了」 | 雜訊，跳過 |
| `WARNING` | 有點異常，但還能運作 | 記著，不急 |
| `ERROR` | 出錯了 | **這才是你要找的** |

### 3. 錯誤訊息直接對應到下一步該查哪裡

| 紀錄裡出現 | 屬於哪一類 | 去哪一章 |
| :--- | :--- | :--- |
| `No such file or directory` | 要用的檔案不存在 | 第 2 章，先確認它到底在不在 |
| `Permission denied` | 權限不足 | 第 3 章 |
| `No space left on device` | 磁碟空間用完 | 第 9 章 |
| `Address already in use` | 連接埠被別的程式佔住 | 第 8 章 |

**這張表是這門課最實用的一張。** 服務為什麼停，答案幾乎都會落在這四類裡面，而紀錄會直接告訴你是哪一類。

## 任務
<br>

| # | 找出什麼 |
| :--- | :--- |
| 1 | `report-job` 這個服務啟動失敗的根本原因是什麼 |

第 1 題要說出根本原因，不是最後一行寫的那句。
