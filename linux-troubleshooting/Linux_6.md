# Stage 6｜追查真相：是誰在敲門？
<br>

## 這一關的處境

服務救回來了。你在清 log 的時候，瞄到 `/var/log/auth.log` 裡有一堆登入失敗的紀錄。

你隨手看了幾行，不同的 IP、不同的帳號、密碼一直錯。

前輩湊過來看了一眼：

> 幾千行，你打算一行一行看嗎？
>
> 這裡有三個問題你要回答：**是誰在敲？敲了幾次？他們想撬開哪個帳號？**
>
> 你不能用眼睛，你要用算的。
>
> 而且記住一件事 —— **不要憑印象猜。你猜錯的地方，就是攻擊者藏身的地方。**

## 你今天只需要知道這些

前面五關，你在**修東西**。這一關開始，你在**查東西** —— 同樣是讀 log，但目的完全不同：

| | 你要找什麼 |
| :--- | :--- |
| **修東西時** | 那一句自白（某個 error） |
| **查東西時** | 規律（誰出現最多次、什麼最頻繁） |

規律不會浮在表面。幾千行紀錄裡，你需要的答案是「203.0.113.45 失敗了 47 次」，但 log 只會給你 47 行分散的紀錄。

所以你要學會一種思考方式：**把一堆文字，變成一張排名表。**

### 管線：把小工具接成一條生產線

Linux 的做法是 —— 每個小工具只做好一件事，你用管線把它們串起來。

```text
   一堆 log
      │
      ▼  grep      撈出我關心的那些行（過濾雜訊）
      │
      ▼  awk       只取出我要的那一欄（例如 IP）
      │
      ▼  sort      把相同的排在一起
      │            ┌──────────────────────────────┐
      │            │ [因] uniq 只能合併「相鄰」的重複 │
      │            │ [果] 所以必須先 sort           │
      │            └──────────────────────────────┘
      ▼  uniq -c   合併相鄰的重複，並數出次數
      │
      ▼  sort -rn  依次數排名，最多的在最上面
      │
   一張排名表
```

> **關於 awk：** 它其實是一個完整的程式語言。
>
> 今天只用它最基本的一招：**把每行按空白切開，取出第 N 欄。**

## 角色 / 工具速查表

| 你想做 | 打這個 |
| :--- | :--- |
| 撈出含某個關鍵字的行 | `grep "Failed password" 檔案` |
| 只取第 11 欄 | `awk '{print $11}'` |
| 一次取兩欄 | `awk '{print $9, $11}'` |
| 把相同的排在一起 | `sort` |
| 數出重複次數 | `uniq -c` |
| 依數字大小排名（大的在前） | `sort -rn` |
| 只看排名前 5 名 | 在管線**最後**接上 `head -n 5` |
| 看命中行的前後文 | `grep -B 3 -A 1 "關鍵字" 檔案` |
| 把權限錯誤訊息丟掉 | 在指令後加上 `2>/dev/null` |

> **怎麼知道 IP 在第幾欄？自己數。**
>
> 把一行拆開，用空白切：
>
> `Jul`(1) `10`(2) `01:22:11`(3) `srv`(4) `sshd[1001]:`(5) `Failed`(6) `password`(7) `for`(8) `root`(9) `from`(10) `203.0.113.45`(11)
>
> **帳號在第 9 欄，IP 在第 11 欄。** 換一種 log，欄位就不一樣 —— 數欄位是這份工作的日常，不用背。

## 跟著做

**一層一層加。每加一層，看它多做了什麼。**

**第 1 層：撈出失敗的登入嘗試。**

`sudo grep "Failed password" /var/log/auth.log`{{execute}}

*→ 成功登入的行被過濾掉了，只剩失敗的。從雜訊裡挑出訊號。*

**第 2 層：只取出 IP。**

`sudo grep "Failed password" /var/log/auth.log | awk '{print $11}'`{{execute}}

*→ 現在只剩一串 IP，其他字都不見了。*

**第 3 層：排序（為了下一步的計數做準備）。**

`sudo grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort`{{execute}}

*→ 相同的 IP 現在相鄰了。*

**第 4 層：計數。**

`sudo grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c`{{execute}}

*→ 每個 IP 前面多了一個數字：它出現幾次。*

**第 5 層：排名。**

`sudo grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -rn`{{execute}}

*→ 一張攻擊來源排名表。最上面那個，就是打你最兇的。*

**換一個問題：他們想撬開哪個帳號？**

管線結構完全不動，只把 `$11` 換成 `$9` ：

`sudo grep "Failed password" /var/log/auth.log | awk '{print $9}' | sort | uniq -c | sort -rn`{{execute}}

*→ 你會看到攻擊者最愛猜哪些帳號。*

**最後一件事：有沒有人真的闖進來了？**

`sudo grep -B 3 -A 1 "Accepted password" /var/log/auth.log`{{execute}}

*→ `-B 3` 是命中行的前 3 行，`-A 1` 是後 1 行。看那次成功登入的前後，夾著什麼。*

## 看懂結果

### 1. 排名表怎麼讀

```text
     47 203.0.113.45   ← 這個 IP 失敗了 47 次，主要攻擊者
      8 198.51.100.7   ← 次要
      1 203.0.113.99   ← 可能只是打錯密碼的同事
```

**次數的差距本身就是情報。** 47 次不是手滑，是自動化工具在跑字典。1 次可能只是有人記錯密碼。

### 2. 前後文的因果判讀

單獨看一行 `Accepted password` 完全正常 —— 每天都有人成功登入。

但如果它**夾在一串失敗嘗試中間**，意義就變了：

```text
Failed password for root ...       ← 失敗
Failed password for admin ...      ← 失敗
Failed password for root ...       ← 失敗
Accepted password for deploy ...   ← 成功！
                                      ⚠️ 可能有人猜中密碼了
```

**這就是為什麼要看前後文。一個事件是否可疑，取決於它周圍發生了什麼。**

> **前輩的報告心法：**
>
> 如果你要把這件事寫進交接報告，不能寫「我覺得怪怪的」，要寫：
>
> *「在該次成功登入之前的短時間內，同一時段出現多次密碼失敗紀錄，符合暴力破解成功的模式，應進一步查證來源 IP 與該帳號的後續行為。」*
>
> **直覺是起點，證據才是交付物。**

## 常見誤會

**「攻擊者一定是在猜 root，那是最高權限。」**

也許。但**攻擊者知道你在盯 root**，所以他們可能改猜 admin、deploy、test。**你的假設，就是他們的藏身處。** 用管線算出來，不要猜。

**「head -n 5 放在管線中間也一樣。」**

差很多。

- ❌ 放**中間** —— 這是**把資料砍掉**。只讓前 5 行流下去，後面的統計建立在殘缺樣本上。結論會是錯的。
- ✅ 放**最後** —— 這是**砍顯示**。統計在完整資料上做完，只是最後只列前 5 名。

**「輸出跑出來了，那就是對的。」**

輸出「看起來合理」不代表你的指令邏輯正確。少了 sort、head 放錯位置、欄位數錯 —— 這些都會產生**看起來很正常、但完全錯誤**的數字。

**一份被截斷過的統計，比沒有統計更危險，因為它看起來有數據支撐。**

## 觀念確認（小任務）

你想統計哪個 IP 登入失敗最多次，打了這條指令，但**少了 sort**：

```text
grep "Failed password" auth.log | awk '{print $11}' | uniq -c | sort -rn
```

結果出現一堆 1 和零星的 2，沒有任何一個 IP 次數很高。最可能的原因是：

- A. 這台機器沒有被攻擊
- B. 少了 sort，相同的 IP 沒有排在一起，uniq -c 數不到它們
- C. awk 的欄位取錯了

**建議答案是 B。** `uniq -c` **只合併相鄰的重複行**。沒有先 sort 把相同的 IP 排在一起，散落各處的同一個 IP 就不相鄰，自然數不出來。**sort 是 uniq -c 的前提，不是可有可無的一步。**
