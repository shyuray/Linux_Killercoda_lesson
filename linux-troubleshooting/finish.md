# 結案

週一早上的那通電話，你處理完了。

服務活著、網站有回應、開機會自動起來、磁碟有空間、權限對了，而且你知道有人正在敲這台機器的門。

---

## 你今天帶走了什麼

**指令會忘。這幾件事不會。**

### 一、一張分診樹

面對「服務不正常」，你不再亂試：

```text
它還活著嗎？ ─▶ 它的遺言說什麼？ ─▶ 是設定、權限、還是資源？ ─▶ 走對應的路
```

**這套方法能處理你沒見過的故障。這才是它真正的價值。**

---

### 二、四個判斷

| 判斷 | 為什麼重要 |
| :--- | :--- |
| **沒報錯不等於做對了** | restart 不報錯，服務可能還躺著。任何操作後都要確認 |
| **「你」讀得到，不代表「它」讀得到** | 服務用自己的低權限帳號跑。判斷權限時，永遠先問「是誰要去讀」 |
| **改設定前先備份，重啟前先驗語法** | 三秒鐘的動作，省你一場事故 |
| **讓資料說話** | 你憑印象猜的地方，就是攻擊者藏身的地方 |

---

### 三、一條可以複製的管線

```text
grep  ─▶  awk  ─▶  sort  ─▶  uniq -c  ─▶  sort -rn
撈出      切欄     排序       計數         排名
```

換掉 grep 的關鍵字和 awk 的欄位，它能回答任何「**誰出現最多次**」的問題：

- 哪個 IP 打我們最兇？
- 哪個帳號被鎖定攻擊？
- 哪個網址被請求最多次？
- 哪個錯誤最常出現？
