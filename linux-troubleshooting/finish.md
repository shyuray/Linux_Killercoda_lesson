# 結案
<br>

週一早上那通電話，你處理完了。

服務恢復執行、網站有回應、開機會自動啟動、磁碟有空間、權限也對了。而且你知道有人正在嘗試登入這台機器。

---
<br>

## 你今天帶走了什麼
<br>

指令會忘，但下面三件事不會。

### 一、一張分診樹

面對「服務不正常」，你不再亂試：

```text
它有沒有在執行？ ─▶ 執行紀錄說了什麼？ ─▶ 是設定、權限、還是資源？ ─▶ 走對應的路
```

**這套方法能處理你沒見過的故障，這才是它真正的價值。**

### 二、四個判斷

| 判斷 | 為什麼重要 |
| :--- | :--- |
| **指令沒報錯，不代表做對了** | restart 不報錯，服務可能還停著。任何操作後都要確認 |
| **你讀得到，不代表服務讀得到** | 服務用的是低權限帳號。判斷權限時，先問「是誰要去讀」 |
| **改設定前先備份，重啟前先驗語法** | 三秒鐘的動作，換掉一場事故 |
| **讓資料說話** | 你憑印象猜的地方，就是攻擊者藏身的地方 |

### 三、一條可以複製的管線

```text
grep  ─▶  awk  ─▶  sort  ─▶  uniq -c  ─▶  sort -rn
撈出      切欄     排序       計數        排名
```

換掉 grep 的關鍵字和 awk 的欄位，它能回答任何「**誰出現最多次**」的問題：哪個來源位址嘗試最多次、哪個帳號被鎖定、哪個網址被請求最多、哪個錯誤最常出現。

## 下一階段
<br>

分診樹上還有一格你沒走過：

```text
   Address already in use  ─▶  連接埠被占用
```

還有磁碟本身怎麼組成、怎麼掛載。那是這台機器對外溝通的方式，以及它腳下的地基。
