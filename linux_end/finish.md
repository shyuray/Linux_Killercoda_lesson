## 你手上現在有什麼
<br>

### 一張完整的分診樹

```text
系統有問題
    │
    ├─ 別人連不到我   →  服務 → log → 設定 / 權限 / 資源 / port
    │                        → 監聽位址 → 防火牆
    │
    ├─ 我連不出去     →  DNS → IP → 路由 → gateway
    │
    └─ 機器很慢       →  top → df -h → df -i
```

**這張圖能處理你沒見過的故障。這才是它的價值。**

指令會忘，Google 一下就回來了。但「面對一個沒見過的問題，該從哪裡下第一刀」——這個沒有地方可以查。

## 六個判斷
<br>

| 判斷 | 為什麼重要 |
| :--- | :--- |
| **指令沒報錯，不代表做對了** | restart 不報錯，服務可能還停著 |
| **你讀得到，不代表服務讀得到** | 服務用的是低權限帳號 |
| **本機測通，不代表外部連得到** | localhost 走 loopback，永遠不出機器 |
| **錯誤訊息不一定會指出「是誰」** | 設定錯給你行號，port 被佔只給你症狀 |
| **「滿了」有兩種** | 空間和 inode，錯誤訊息一模一樣 |
| **讓資料說話** | 你憑印象猜的地方，就是問題藏身的地方 |

## 三個紀律
<br>

**改設定前先備份。** 你不需要記得自己改了什麼，你只需要能還原。

**套用前先驗證。** nginx 用 `nginx -t`、fstab 用 `mount -a`、netplan 用 `netplan try`。**改的是開機或連線必經的路，就一定要有這一步。**

**動手前先問：這一步會不會把自己鎖在外面。** `ufw enable` 之前先 `allow 22`。

## 一條可以複製的管線
<br>

```text
撈出來  →  切出要的欄位  →  排序  →  計數  →  排名
grep         awk            sort    uniq -c   sort -rn
```

你用它算過攻擊來源 IP 的排名，也用它找過哪個目錄檔案數最多。

**換掉前面撈的東西，它就能回答不同的問題。** 你學的不是一條指令，是一個模式。

## 最後
<br>

這兩個階段，你沒有背過任何東西。

你做的是：看到症狀 → 找證據 → 縮小範圍 → 找出原因 → 修好 → 確認。

**下次遇到一個完全沒見過的服務、完全沒見過的錯誤訊息，你還是可以用同一套流程。**

因為你學的不是 nginx，是怎麼查。
