# Linux 系統除錯實戰（二）：連不到與磁碟
<br>

## 上次沒走完的那兩格
<br>

上一階段你救回了公司的內部系統，也學會了一套方法：先確認服務有沒有在執行，再讀 log，然後判斷是設定、權限、還是資源。

但那張分診樹上，還有一格是空的：

```text
Address already in use  ─▶  port 被佔用  ─▶  ？
```

還有一整類問題，你上次連碰都沒碰到：**服務明明在執行、設定沒錯、權限沒錯、磁碟沒滿，但別人就是連不到。**

這一階段就是把這兩件事補完。

---
<br>

## 今天你要學會什麼
<br>

- 服務在跑，為什麼還是連不到？
- 這台機器跟外界之間，中間有哪幾層？每一層怎麼查？
- 磁碟明明還有空間，為什麼寫不進去？

---
<br>

## 你先知道這個就好
<br>

**IP 位址**是一個數字，用來識別網路上的一台機器。你的機器有 IP，資料庫那台有 IP，Google 也有。

但一台機器上同時跑著網頁、資料庫、SSH。資料送到這台機器之後，作業系統核心要決定：**這筆資料該交給哪一個程式？**

解法是 **port**。每一筆網路資料除了 IP，還帶一個 port 編號。核心內部有一張對應表：

```text
port 22  → sshd
port 80  → nginx
port 53  → systemd-resolved
```

資料進來，核心讀 port 編號，查表，交給對應的程式。

哪個服務用哪個 port 是**約定**，不是規定：80 是 HTTP、443 是 HTTPS、22 是 SSH、3306 是 MySQL。約定的目的，是讓別人不用問就知道該連哪裡。

**這張對應表，一個 port 只能有一筆紀錄。** 記住這句，Stage 1 就是它。

## 這一階段的分診樹
<br>

「連不到」有兩個方向，症狀可能一模一樣，但排查方向完全相反。

```text
                    「連不到」
                         │
        ┌────────────────┴────────────────┐
        ▼                                 ▼
  別人連不到我                        我連不出去
        │                                 │
  服務在執行嗎？                     名稱換得到 IP 嗎？
  systemctl status                   curl 名稱 vs curl IP
        │                                 │
  有在監聽嗎？監聽哪個位址？          有 IP、有路由嗎？
  sudo ss -tlnp                      ip a  /  ip r
        │                                 │
  防火牆允許嗎？                     gateway 通嗎？
  sudo ufw status                    ping -c 3 <gateway>
```

**第一個問題永遠是：是誰連不到誰。** 問錯方向，後面每一步都白做。

## 排查守則
<br>

跟上一階段一樣，這四句仍然算數。

1. **先看證據，再下結論。**
2. 只相信指令實際查到的狀態。
3. 錯誤訊息通常已經指出問題在哪，只是它不一定會指出**是誰**造成的。
4. **指令沒有報錯，不代表事情做對了。**

## 暖身：先確認環境就緒
<br>

這個環境需要幾十秒佈置。**先執行下面這行，等它印出「環境就緒」再往下做**，否則你會看到一台還沒佈置完的機器。

`while [ ! -f /tmp/scenario-ready ]; do echo "環境準備中，請稍候…"; sleep 3; done; echo "環境就緒，可以開始"`{{execute}}

就緒之後，先看這台機器有哪些 port 正在被使用：

`sudo ss -tlnp`{{execute}}

> `ss` 顯示網路連線狀態。四個參數：`-t` 只看 TCP、`-l` 只看正在監聽的、`-n` 用數字顯示 port 不轉成名稱、`-p` 顯示是哪個程式。
>
> **`-p` 需要 sudo。** 它只顯示你自己執行的程序，而服務多半是 root 執行的。少了 sudo，那一欄會是空的——它不會報錯，只是資料缺一塊。

## 觀念確認（小任務）
<br>

同事說「網頁打不開」。你問的第一個問題應該是：

- A. 服務有沒有掛掉
- B. 是誰連不到誰
- C. 是不是網路壞了

**建議答案是 B。** 「我從外面連不到你們的網站」和「這台伺服器連不到外網」是兩個完全相反的方向，工具和順序都不一樣。問錯方向，後面每一步都在做白工。
